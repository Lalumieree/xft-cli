# 第三方对接回调批量审批单据

## 接口调用地址 
 生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/bills/thirdPartCallback 
 测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/bills/thirdPartCallback 
 ## 接口描述 
费控系统中“推送外部系统”节点会将单据信息推送给外部系统，此时费控中单据会在该审批节点等待，外部系统审批后，需要调用该接口来处理费控中单据的审批状态 
## 调用方式 
 METHOD: POST  
 CONTENT-TYPE: application/json 
## 接口输入参数 
 
| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| header | OBJECT |  | 请求头根节点 |  |

| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| body | OBJECT | 是 | 请求体根节点 |  |
| body.billList | ARRAY | 是 | 单据审批结果信息 |  |
| body.billList.billId | LONG | 是 | 单据编号 | 2024041714594536 |
| body.billList.status | STRING | 是 | 第三方处理状态：1-审批通过；3-拒绝；B-退回 | 1 |
| body.billList.remark | STRING | 否 | 单据状态为拒绝、退回时，需要传入单据审核备注信息，未传备注信息会出现如下报错：拒绝、退回的单据审核备注信息不能为空。 | 通过 |
| body.billList.backTypeFlag | INTEGER |  | 0-退回后无需走已审批节点；1-需重走全部节点，默认为0 | 0 |
| body.billList.backNodeId | STRING |  | 传空时默认退回至发起人<br>指定退回节点，退回时则不回到发起人<br>退回节点可使用审批历史获得<br>退回节点需要在流程历史中最新一次重新提交后的节点范围内 |  |

| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| query | OBJECT |  | query根节点 |  |
| query.OPAUID | STRING | 是 | 薪福通企业号 | XFT00001 |
 
## 接口输出参数
 
| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| header | OBJECT |  | 响应头根节点 |  |

| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| body | OBJECT |  | 响应体根节点 |  |
| body.data | OBJECT | 否 | 返回数据 |  |
| body.data.failedCnt | LONG | 否 | 失败个数 | 2 |
| body.data.total | LONG | 否 | 处理总数 | 2 |
| body.data.detail | ARRAY | 否 | 处理详情 |  |
| body.data.detail.billId | LONG | 否 | 单据编号 | 2022000000000000 |
| body.data.detail.result | STRING | 否 | 1-成功；0-失败 | 0 |
| body.data.detail.failedReason | STRING | 否 | 失败原因 | 找不到对应审批任务 |
| body.success | BOOLEAN | 否 | 调用成功或失败 | true |
| body.message | STRING | 否 | 返回调用信息 | 回调成功 |
 
## 接口调用示例 
 ### 请求示例
`{BaseURL}/api/v1/bills/thirdPartCallback?OPAUID=XFT00001`
```json
{
    "billList": [
        {
            "billId": 2022000000000000,
            "status": "1"
        },
        {
            "billId": 2022000000000001,
            "status": "1"
        }
    ]
}
```
### 返回示例
```json
{
    "data": {
        "failedCnt": 2,
        "total": 2,
        "detail": [
            {
                "billId": 2022000000000000,
                "result": "0",
                "failedReason": "找不到对应审批任务"
            },
            {
                "billId": 2022000000000001,
                "result": "0",
                "failedReason": "找不到对应审批任务"
            }
        ]
    },
    "success": true,
    "message": "回调成功"
}
```
