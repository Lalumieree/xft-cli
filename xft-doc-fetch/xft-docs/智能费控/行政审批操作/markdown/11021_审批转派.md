# 审批转派

## 接口调用地址 
 生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/approve/dispatch 
 测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/approve/dispatch 
 ## 接口描述 
单据审批转派 
## 调用方式 
 METHOD: POST  
 CONTENT-TYPE: application/json 
## 接口输入参数 
 
| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| header | OBJECT |  | 请求头根节点 |  |

| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| body | OBJECT |  | 请求体根节点 |  |
| body.businessList | ARRAY | 是 | 单次最多处理50条审批信息 |  |
| body.businessList.approveCategory | STRING | 否 | 目前支持单据(BILL),默认不传为单据 | BILL |
| body.businessList.approveOpinion | STRING | 否 | 根据审批节点的审批意见配置进行校验，当审批意见配置为：所有操作必填时，该值为必填 | OK |
| body.businessList.businessEmpIdList | ARRAY | 是 | 1. 审批对应的转派员工,必须存在且只能为一人。2. 目前转派 多个businessId 对应单个用户，用户放在集合第一个元素中即可 | ["W0000"] |
| body.businessList.businessId | LONG | 是 | 单据ID/流程编号 | 2022062101309617 |
| body.empId | STRING | 是 | 发起审批操作人的员工编号 | U0000 |

| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| query | OBJECT |  | query根节点 |  |
| query.OPAUID | STRING | 是 | 薪福通企业号 | XFT00001 |
 
## 接口输出参数
 
| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| header | OBJECT |  | 响应头根节点 |  |

| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| body | OBJECT |  | 响应体根节点 |  |
| body.error | STRING | 否 | 记录失败的数据, 值为key:错误原因。当数据整体错误时,返回 errorMessage:错误原因 | null |
| body.success | BOOLEAN | 否 | 执行结果布尔值:<br>成功为true;有一条失败则为false | true |
| body.successKey | ARRAY | 否 | 成功的数据会返回成功的唯一标识key | [2022062101309617] |
 
## 接口调用示例 
 ###请求示例
` {BaseURL}/api/v1/approve/dispatch?OPAUID=XFT00001
```json
{
  "businessList": [
    {
      "approveCategory": "BILL",
      "approveOpinion": "转派",
      "businessEmpIdList": ["W0000"],
      "businessId": 2022062101309617
    }
  ],
  "empId": "U0000"
}
```
### 返回示例
```json
{
	"error": "",
	"success": true,
	"successKey": [2022062101309617]
}
```
