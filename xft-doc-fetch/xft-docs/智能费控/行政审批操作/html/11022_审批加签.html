审批加签

接口调用地址

生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/approve/countersign

测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/approve/countersign

接口描述

单据审批加签

调用方式

METHOD: POST

CONTENT-TYPE: application/json

接口输入参数

参数类型必填说明示例
headerOBJECT请求头根节点
参数类型必填说明示例
bodyOBJECT请求体根节点
body.businessListARRAY是审批加签目前仅支持单个处理
body.businessList.approveCategorySTRING否目前支持单据(BILL),默认不传为单据BILL
body.businessList.approveOpinionSTRING否根据审批节点的审批意见配置进行校验，当审批意见配置为：所有操作必填时，该值要求必填加签
body.businessList.businessEmpIdListARRAY是审批加签,加签人员工编号不能重复且加签人员工编号不能包含本人["W000"]
body.businessList.businessIdLONG是单据ID/流程编号2022062101309617
body.businessList.signTypeSTRING是PARALLEL 或签、会签、平行加签，FRONT 在我之前，BEHIND 在我之后。默认为FRONT 在我之前FRONT
body.empIdSTRING是发起审批操作人的员工编号U0000
参数类型必填说明示例
queryOBJECTquery根节点
query.OPAUIDSTRING是薪福通企业号XFT00001

接口输出参数

参数类型必填说明示例
headerOBJECT响应头根节点
参数类型必填说明示例
bodyOBJECT响应体根节点
body.errorSTRING否记录失败的数据, 值为key:错误原因。当数据整体错误时,返回 errorMessage:错误原因null
body.successBOOLEAN否执行结果布尔值:
成功为true;有一条失败则为false		true
body.successKeyARRAY否成功的数据会返回成功的唯一标识key[2022062101309617]
