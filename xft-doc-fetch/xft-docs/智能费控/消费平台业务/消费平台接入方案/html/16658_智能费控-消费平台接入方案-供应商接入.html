薪福通费控消费平台与外部供应商集成方案

2、前置准备

前置准备主要指外部机酒平台需要提供给薪福通的内容及准备工作，具体包括：

序号事项描述信息备注
1开放平台相关对接流程供应商系统需要基于应用接入开放平台的开放API，具体可参考开发部分的前置说明，同时需获得相应的文档权限

开放平台地址：[https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=10684 ](#/doc/open-document?id=10692&mid=10684)，开放平台相关问题可联系分行相关负责人
2企业开通申请事件订阅&开通结果回调供应商通过开放平台订阅企业开通申请事件，并回调开通结果回调接口
3员工/部门查询接口&员工/部门事件订阅供应商调用开放平台员工/部门查询接口获取员工/部门信息，并通过订阅员工/部门事件获取信息变更
4申请单事件订阅供应商通过开放平台订阅申请单推送事件，并调用申请单推送回调，返回推送结果
5单点登录供应商系统在开放平台注册应用，配置单点登录地址，调用门户接口，费控消费平台直接访问供应商所配的单点登录地址即可，该部分主要为供应商与门户交互
6订单同步供应商调用开放平台订单保存接口推送订单分为机酒火打车用餐分别进行推送

3、开发模式

3.1、整体流程

![image(1).png](https://s3gw.cmbchina.com/lw36.01-public-prd/public-86274e5f-78b3-4746-85b9-cda628e6fa77)

开发工作可以在UAT2或者生产进行，供应商通过开放平台的应用调用开放平台的事件或接口开发，联通工作可以在UAT2环境完成，以下文档以在UAT2进行开发进行介绍。

3.2、开发过程

3.2.1、开放平台API前置准备

企业与开放平台API对接流程说明：企业需在测试环境和生产环境分别注册企业，并在开放平台的开发者后台中注册应用，以满足UAT2联调和生产联调，详细的对接流程可参考以下文档：<u>[https://xft.cmbchina.com/open/#/doc/open-document?id=10631&mid=11539](https://xft.cmbchina.com/open/#/doc/open-document?id=10631&mid=11539，对接的详细问题可咨询分行相关负责人)</u>[，对接的详细问题可咨询分行相关负责人](https://xft.cmbchina.com/open/#/doc/open-document?id=10631&mid=11539，对接的详细问题可咨询分行相关负责人)

如企业无接口文档权限，需在生产环境注册企业并获得文档权限，并访问行外平台相对应的文档

事件订阅接口参考以下文档：

<https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=10686>

3.2.2、企业开通申请

企业在费控第三方开通页面输入账号绑定相关信息后，费控将进行开通申请事件发布，供应商通过开放平台订阅该事件获取相关信息进行相关处理，并将处理结果通过企业开通结果回调接口返回，费控获取该结果用于判断企业是否成功开通。

消费平台开通申请事件：

<https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=15549>

消费平台开通结果回调接口：

<https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=15516>

3.2.3、员工/部门

员工部门数据同步通过调用开放平台员工部门查询接口实现，并通过订阅相关事件获取新增、变更、删除等信息。

部门查询接口文档：

<https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=11620>

员工查询接口文档：

<https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=12330>

订阅部门/员工新增、变更、删除相关事件，事件清单：

<https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=11233>

3.2.4、申请单推送

根据企业在消费平台->设置规则配置的申请单推送类型、推送节点，通过事件发布的形式异步进行申请单推送，供应商订阅事件获取申请单信息，并将结果通过回调接口告知费控。

第三方申请单推送事件：

<https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=13761>

费控推送申请单回调接口：

<https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=13170>

单点登录采用薪福通门户的平台统一单点登录，需要第三方在开放平台注册应用，配置单点登录地址，调用门户接口，下面以之前薪福通福利系统与供应商系统对接图为例：

门户单点登录接口相关文档:

[ ](#/doc/open-document?id=10692&mid=12581)<https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=12581>

订单同步需要供应商调用开放平台的接口，分为酒店、火车、飞机、餐食、用车接口

订单保存接口（餐食为例）：

<https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=12464>

注意，外部平台订单同步至薪福通，需要对城市信息进行映射，薪福通费控城市查询接口：

<https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=11044>

* 数据同步时机\n主要包括人员、部门同步以及订单推送的触发机制。参考模板如下：

场景开通触发变更事件触发下单后触发定时任务时间<br><br> （若通过定时任务触发）触发规则详情
人员同步是是\每天2:30
组织同步是是\每天2:00
订单推送\\是每天3:00例：下单后会进行实时推送&每天凌晨3点推送前三天的订单
