# 3、开发模式

## 3.1、整体流程

![开发流程.png](https://s3gw.cmbchina.com/lw36.01-public-prd/public-8d0f12f1-53ca-4854-a3d8-37d76f46d352)

开发工作可以在沙盒环境或者生产进行，供应商通过开放平台的应用调用开放平台的事件或接口开发，联调工作可以在沙盒环境完成，以下文档以在沙盒进行开发进行介绍。

## 3.2、开发过程

### 3.2.1、开放平台API前置准备

企业与开放平台API对接流程说明：企业需在测试环境和生产环境分别注册企业，并在开放平台的开发者后台中注册应用，以满足沙盒环境联调和生产联调，详细的对接流程可参考以下文档：<https://xft.cmbchina.com/open/#/doc/open-document?id=10631&mid=11090>[，](https://xft.cmbchina.com/open/#/doc/open-document?id=10631&mid=11539，对接的详细问题可咨询@安晋峰/80307077)对接的过程中的问题咨询可联系分行支持人员。

如企业无接口文档权限，需在生产环境注册企业并获得文档权限，并访问行外平台相对应的文档

事件订阅接口参考以下文档：

<https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=10686>

### 3.2.2、员工/部门

员工部门数据同步通过调用开放平台员工部门查询接口实现，并通过订阅相关事件获取新增、变更、删除等信息。

部门查询接口文档：

<https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=11620>

员工查询接口文档：

<https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=12330>

订阅部门/员工新增、变更、删除相关事件，事件清单：

<https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=11233>

### 3.2.3、申请单推送

员工在智能费用提单，单据信息会通过事件发布的形式推送到外部，供应商通过网页应用订阅事件，根据单据类型和编号过滤获取申请单信息，并根据申请单编号，调用差旅申请单详情接口，查询获取单据详细信息。

单据推送事件：

<https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=15408>
差旅申请单详情查询接口：
<https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=13404>

## 3.3、测试过程

## 3.4、全流程验证

![验证流程.png](https://s3gw.cmbchina.com/lw36.01-public-prd/public-646b9b7c-1332-4262-bd56-dc3af766c31a)

1、前置功能验证：员工、组织架构信息同步

2、预定流程验证：员工登录薪福通，在智能费控系统提交申请单（审批），员工点击网页应用跳转商旅平台，选择申请单进行预定

3、预定后订单数据验证：员登录薪福通，在智能费控-消费平台-订单中心，可查看订单信息
