# 导入预算科目信息

## 接口调用地址 
 生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/budget/import/subject 
 测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/budget/import/subject 
 ## 接口描述 
支持外部系统通过接口导入预算科目信息进入费控，不支持部分成功 
## 调用方式 
 METHOD: POST  
 CONTENT-TYPE: application/json 
## 接口输入参数 
 
| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| root | OBJECT |  | 请求头根节点 |  |

| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| root | OBJECT |  | 请求体根节点 |  |
| root.opsEmpNumber | STRING | 是 | 填写员工花名册编码 | 001 |
| root.subjectTypes | ARRAY | 否 | 传入则进行导入预算科目分类<br>最多支持传入100条 |  |
| root.subjectTypes.budgetSubTypeId | STRING | 是 | 仅支持数字和字母 | z001 |
| root.subjectTypes.budgetSubTypeName | STRING | 是 | 预算科目分类名称 | api导入科目分类 |
| root.subjects | ARRAY | 是 | 最多支持传入100条 |  |
| root.subjects.expIds | STRING | 是 | 填写费用类型编码，以英文逗号,分割 | 01,02 |
| root.subjects.subName | STRING | 是 | 预算科目名称 | api测试科目名称1-1 |
| root.subjects.subTypeId | STRING | 是 | 仅支持数字和字母 | z001 |

| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| root | OBJECT |  | query根节点 |  |
 
## 接口输出参数
 
| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| root | OBJECT |  | 响应头根节点 |  |

| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| root | OBJECT |  | 响应体根节点 |  |
| root.data | OBJECT | 否 | 接口返回导入结果 |  |
| root.data.success | BOOLEAN | 否 | true说明导入成功<br>false说明导入失败 | false |
| root.data.errors | ARRAY | 否 | 如果导入失败，这里会返回具体的导入失败原因<br>如果是导入成功，则为这里为空 |  |
| root.data.errors.type | STRING | 否 | 预算科目分类编码<br>预算科目分类名称<br>预算科目名称<br>允许占用的费用/补贴类型 | 预算科目分类编码 |
| root.data.errors.index | LONG | 否 | 导入失败的科目序号 | 1 |
| root.data.errors.msg | ARRAY | 否 | 导入失败原因列表，元素为具体失败原因字符串 | ["预算科目分类名称【api导入科目分类1】已存在","预算科目分类名称【api导入科目分类1】有重复项"] |
| root.success | BOOLEAN | 否 | true说明接口调用成功<br>false说明接口调用失败 | true |
| root.message | STRING | 否 | 接口调用返回信息 | 调用成功！ |
 
## 接口调用示例 
 #### 请求报文
```json
{
  "opsEmpNumber": "2",
  "subjectTypes": [
    {
      "budgetSubTypeId": "z001",
      "budgetSubTypeName": "api导入科目分类1"
    }
  ],
  "subjects": [
    {
      "expIds": "01,02",
      "subName": "api测试科目名称1-1",
      "subTypeId": "z001"
    }
  ]
}
```
#### 响应报文
#### 接口调用失败的报文，示例仅为其中一部分失败样例
```json
{
    "success": false,
    "message": "导入的预算科目分类和预算科目为空"
}
```
#### 导入成功的报文
```json
{
    "data": {
        "success": true,
        "errors": []
    },
    "success": true,
    "message": "调用成功！"
}
```
#### 导入失败的报文，示例仅为其中一部分失败样例
```json
{
    "data": {
        "success": false,
        "errors": [
            {
                "type": "预算科目分类编码",
                "index": 1,
                "msg": [
                    "预算科目分类编码【z001】已存在"
                ]
            }
        ]
    },
    "success": true,
    "message": "调用成功！"
}
```  
## 错误码说明 
外层success为接口调用状态 true为成功 其余都是失败
data内的sucess为实际导入成功状态， true为成功 其余都是失败

常见报错如下：
对于导入的预算科目分类列表：
预算科目分类编码【xx】已存在
预算科目分类编码【xx】有重复项
预算科目分类名称【xx】已存在
预算科目分类名称【xx】有重复项
预算科目分类名称不能为空
预算科目未配置费用或补助

对于导入的预算科目列表：
允许占用的费用/补贴类型【xx】不存在
同一个科目不允许存在相同的费用/补贴类型
预算科目名称【xx】已存在
预算科目名称【xx】有重复项
预算科目名称不能为空
科目所属分类【xx】不存在
