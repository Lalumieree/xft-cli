# 编辑预算父方案

## 接口调用地址 
 生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/budget/edit-budget-parent-plan 
 测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/budget/edit-budget-parent-plan 
 ## 接口描述 
本接口适用于单条修改预算方案信息
调用本接口后可在“智能费控-系统设置-预算设置-预算方案“中查看修改后的预算方案信息。 
## 调用方式 
 METHOD: POST  
 CONTENT-TYPE: application/json 
## 接口输入参数 
 
| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| root | OBJECT |  | 请求头根节点 |  |

| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| root | OBJECT |  | 请求体根节点 |  |
| root.allowExternalSystemBill | STRING | 是 | 0-不允许<br>1-允许 | 1 |
| root.allowMonthOrderBudgetOccupy | STRING | 是 | 0-不支持<br>1-支持 | 1 |
| root.budgetApproveTime | STRING | 是 | NONE-不需要审批<br><br>WHEN_INIT-期初预算编制需审批<br><br>WHEN_MODIFY-预算追加调整需审批<br><br>ANYTIME-期初预算编制/预算追加调整均需审批 | NONE |
| root.budgetEndDate | STRING | 是 | 格式为yyyy-MM-dd | 2025-12-31 |
| root.budgetName | STRING | 是 | 预算方案名称 | 测试预算方案导入 |
| root.budgetOrgs | ARRAY | 否 | 预算组织自增id列表，最多支持关联100个预算组织<br>如果不填，则默认为支持所有的预算组织 | [1,2,3] |
| root.budgetStartDate | STRING | 是 | 格式为yyyy-MM-dd | 2025-01-01 |
| root.budgetSubs | ARRAY | 否 | 预算科目自增id列表，最多支持关联100个预算科目<br>如果不填，则默认为支持所有的预算科目 | [1,2,3] |
| root.eventIds | ARRAY | 否 | 表单自增id列表，最多支持关联100个表单<br>如果不填，则默认为适用所有的表单 | [1,2,3] |
| root.needPostAccount | STRING | 是 | 0-无需过账<br>1-需要过账<br>若设置过账后生效，预算单审批通过后需点击过账后预算才能生效<br>预算是否需要审批为NONE时，默认设置无需过账-0 | 0 |
| root.opsEmpNumber | STRING | 是 | 操作人花名册编码 | 001 |
| root.budgetParentPlanSeqId | LONG | 是 | 根据方案自增id匹配预算方案信息进行更新 | 100 |

| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| root | OBJECT |  | query根节点 |  |
 
## 接口输出参数
 
| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| root | OBJECT |  | 响应头根节点 |  |

| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| root | OBJECT |  | 响应体根节点 |  |
| root.returnCode | STRING | 否 | SUC0000-成功<br>E0001-系统错误<br>E0002-参数错误 | SUC0000 |
| root.success | BOOLEAN | 否 | true说明接口调用成功<br>false说明接口调用失败 | true |
| root.message | STRING | 否 | 接口调用返回信息 | 成功 |
 
## 接口调用示例 
 ### 请求报文
```json
{
    "budgetParentPlanSeqId":100,
    "allowExternalSystemBill": "0",
    "allowMonthOrderBudgetOccupy": "0",
    "budgetApproveTime": "NONE",
    "budgetEndDate": "2026-12-31",
    "budgetName": "测试预算方案导入1",
    "budgetOrgs": [1],
    "budgetStartDate": "2024-01-01",
    "budgetSubs": [1],
    "eventIds": [1],
    "needPostAccount": "0",
    "opsEmpNumber": "001"
}
```
### 更新成功的响应报文
```json
{
    "body": "",
    "returnCode": "SUC0000",
    "success": true,
    "message": "成功"
}
```
### 更新失败的响应报文，示例仅为其中一部分失败样例
```json
{
    "returnCode": "E0001",
    "success": false,
    "message": "业务错误: 预算方案不存在，请检查后重试！"
}
```
### 接口调用失败的报文，示例仅为其中一部分失败样例
```json
{
    "code": "E0002",
    "message": "预算方案自增ID不能为空;"
}
```  
## 错误码说明 
sucess为更新成功状态， true为成功 其余都是失败
| 常见报错|
| --- |
| 正在进行预算额度导入任务，请稍后再试|
| 预算方案不存在，请检查后重试！|
| 预算方案配置了固定区间，不允许修改预算期间，请检查|
| 该方案下还有待审批的预算，暂不能修改审批设置|
| 当前预算方案日期设置错误，请检查|
| 预算组织在同一时间不能配置于多个方案|
| 预算科目在同一时间不能配置于多个方案|
| 预算组织科目在同一时间不能配置于多个方案|
| 预算区间只支持扩大，不允许缩短|
|已经有预算发生额，预算区间只能向后扩展|
