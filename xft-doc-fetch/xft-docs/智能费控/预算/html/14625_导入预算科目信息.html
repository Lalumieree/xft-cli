导入预算科目信息

接口调用地址

生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/budget/import/subject

测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/budget/import/subject

接口描述

支持外部系统通过接口导入预算科目信息进入费控，不支持部分成功

调用方式

METHOD: POST

CONTENT-TYPE: application/json

接口输入参数

参数类型必填说明示例
rootOBJECT请求头根节点
参数类型必填说明示例
rootOBJECT请求体根节点
root.opsEmpNumberSTRING是填写员工花名册编码001
root.subjectTypesARRAY否传入则进行导入预算科目分类
最多支持传入100条
root.subjectTypes.budgetSubTypeIdSTRING是仅支持数字和字母z001
root.subjectTypes.budgetSubTypeNameSTRING是预算科目分类名称api导入科目分类
root.subjectsARRAY是最多支持传入100条
root.subjects.expIdsSTRING是填写费用类型编码，以英文逗号,分割01,02
root.subjects.subNameSTRING是预算科目名称api测试科目名称1-1
root.subjects.subTypeIdSTRING是仅支持数字和字母z001
参数类型必填说明示例
rootOBJECTquery根节点

接口输出参数

参数类型必填说明示例
rootOBJECT响应头根节点
参数类型必填说明示例
rootOBJECT响应体根节点
root.dataOBJECT否接口返回导入结果
root.data.successBOOLEAN否true说明导入成功
false说明导入失败		false
root.data.errorsARRAY否如果导入失败，这里会返回具体的导入失败原因
如果是导入成功，则为这里为空
root.data.errors.typeSTRING否预算科目分类编码
预算科目分类名称
预算科目名称
允许占用的费用/补贴类型		预算科目分类编码
root.data.errors.indexLONG否导入失败的科目序号1
root.data.errors.msgARRAY否导入失败原因列表，元素为具体失败原因字符串["预算科目分类名称【api导入科目分类1】已存在","预算科目分类名称【api导入科目分类1】有重复项"]
root.successBOOLEAN否true说明接口调用成功
false说明接口调用失败		true
root.messageSTRING否接口调用返回信息调用成功！

接口调用示例

请求报文


{
  "opsEmpNumber": "2",
  "subjectTypes": [
    {
      "budgetSubTypeId": "z001",
      "budgetSubTypeName": "api导入科目分类1"
    }
  ],
  "subjects": [
    {
      "expIds": "01,02",
      "subName": "api测试科目名称1-1",
      "subTypeId": "z001"
    }
  ]
}

响应报文

接口调用失败的报文，示例仅为其中一部分失败样例


{
    "success": false,
    "message": "导入的预算科目分类和预算科目为空"
}

导入成功的报文


{
    "data": {
        "success": true,
        "errors": []
    },
    "success": true,
    "message": "调用成功！"
}

导入失败的报文，示例仅为其中一部分失败样例


{
    "data": {
        "success": false,
        "errors": [
            {
                "type": "预算科目分类编码",
                "index": 1,
                "msg": [
                    "预算科目分类编码【z001】已存在"
                ]
            }
        ]
    },
    "success": true,
    "message": "调用成功！"
}

错误码说明

外层success为接口调用状态 true为成功 其余都是失败

data内的sucess为实际导入成功状态， true为成功 其余都是失败

常见报错如下：

对于导入的预算科目分类列表：

预算科目分类编码【xx】已存在

预算科目分类编码【xx】有重复项

预算科目分类名称【xx】已存在

预算科目分类名称【xx】有重复项

预算科目分类名称不能为空

预算科目未配置费用或补助

对于导入的预算科目列表：

允许占用的费用/补贴类型【xx】不存在

同一个科目不允许存在相同的费用/补贴类型

预算科目名称【xx】已存在

预算科目名称【xx】有重复项

预算科目名称不能为空

科目所属分类【xx】不存在
