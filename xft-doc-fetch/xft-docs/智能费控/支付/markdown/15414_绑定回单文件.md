# 绑定回单文件

## 接口调用地址 
 生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/receipt-file/bind 
 测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/receipt-file/bind 
 ## 接口描述 
回单文件绑定接口，传入文件唯一编号，可结合回单文件上传接口使用。
支持传入billId+payIndex 或者 只传 oboSeqId，作为明细的匹配值,
billId+payIndex 和 oboSeqId 都传时，明细匹配值要保持一致。
最多支持传入100条回单文件key。
多次调用绑定接口，回单文件绑定为累加，原有绑定的回单文件有效。
回单文件绑定时，无效文件key会自动清除，只保留有效文件key。
支付明细已绑定的文件key再次进行绑定不进行重复绑定。
 
## 调用方式 
 METHOD: POST  
 CONTENT-TYPE: application/json

| root | OBJECT |  | 请求体根节点 |  |
| root.billId | LONG |  | 支持传入billId+payIndex 或者 只传 oboSeqId，作为明细的匹配值 | 2025031804139 |
| root.payIndex | INTEGER |  | 支持传入billId+payIndex 或者 只传 oboSeqId，作为明细的匹配值 | 1 |

| root.receiptList | ARRAY | 是 | 最多支持传入100条回单文件key | [ "903e38be-01c7-4419-992d-18092db585b0" ] |

| 参数 | 类型 | 必填 | 说明 | 示例 |

## 接口调用示例 
 ### 请求报文
```json
{
    "billId": 2025011100018011,
    "payIndex": "1",
    "oboSeqId": 1,
    "receiptList": [
        "c35397d6-5234-4fb2-b874-625951f475c9",
        "c35397d6-5234-4fb2-b874-625951f47517"
    ]
}
```
### 响应报文
```json
{
    "success": true,
    "errorMessage": ""
}
```
