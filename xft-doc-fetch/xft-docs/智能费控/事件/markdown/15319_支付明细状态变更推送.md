# 支付明细状态变更推送

## 基本信息 
 
 事件名称: 支付明细状态变更推送 
 事件编号: ITRIPPAYMODIFY  
## 事件描述 
 智能费控支付明细状态发生变更后，推送支付明细状态变更事件
注意事项：
1、推送不保证消息前后顺序：
这意味着系统推送的状态变更通知不一定会严格按照实际发生支付时间顺序来进行。因此，在接收这类通知时，需要考虑到信息可能不会按逻辑顺序呈现，用户可以通过支付明细查询的api接口自行确认最新的状态。
2、线上支付状态同步延迟性：
目前线上支付状态的同步存在三种途径：
1）中台消息推送
2）用户列表手动同步状态
3）次日凌晨全量跑批
这意味着虽然支付已经完成，但相关的通知消息可能不会立即到达（最迟到次日03点）
3、每批次上限100条数据：
系统一次最多能推送100条支付明细状态变更。如果需要发送的通知数量超过此限制，会分批发送。这有助于确保每次传输的数据量不至于过大，影响系统的稳定性和效率。 
## 触发情况
 发起支付、同步线上渠道支付状态、取消支付、手动确认支付、退回待支付等操作变更支付明细状态时，触发该事件
