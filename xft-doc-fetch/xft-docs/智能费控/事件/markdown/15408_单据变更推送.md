# 单据变更推送

## 基本信息 
 
 事件名称: 单据变更推送 
 事件编号: ITRIPBILLALTER  
## 事件描述 
 智能费控单据状态发生变更（可能是从待审核变为已审核，或者从待支付变为已支付等）完成后，向指定的相关人员或系统发送通知，告知单据状态的变化情况。
注意事项
1、推送不保证消息前后顺序：
这意味着系统推送的状态变更通知不一定会严格按照实际发生的审批与支付时间顺序来进行。因此，在接收这类通知时，需要考虑到信息可能不会按逻辑顺序呈现，用户需自行确认最新的状态。
2、支付成功推送存在延迟性：
支付成功的信息可能会出现延迟推送的情况。这意味着虽然支付已经完成，但相关的通知消息可能不会立即到达（最迟到次日03点）。
3、每批次上限1000条数据：
系统一次最多能推送1000条数据变更。如果需要发送的通知数量超过此限制，会分批发送。这有助于确保每次传输的数据量不至于过大，影响系统的稳定性和效率。 
## 触发情况
 
场景1：提单、审批完结、审批否决、退回至开始节点、撤回后删除、完结废弃

场景2：支付成功
