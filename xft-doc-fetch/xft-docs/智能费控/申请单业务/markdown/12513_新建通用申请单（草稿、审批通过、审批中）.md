# 新建通用申请单（草稿、审批通过、审批中）

## 接口调用地址 
 生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/bills/apply/common/create 
 测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/bills/apply/common/create 
 ## 接口描述 
支持调用此接口新增草稿、审批通过的通用申请单

### 自定义字段传值格式和说明如下：

| 字段类型 | 说明 | fieldValue示例 |
| --- | --- | --- |
| 单行文本 | 长度限制以自定义字段配置为准，value以字符串格式传入 | 单行文本11 |
| 多行文本 | 长度限制以自定义字段配置为准，value以字符串格式传入 | 多行文本11 |

| 下拉选择 | 支持单/多选，以自定义字段配置为准, value 以选项值传入 Json格式字符串 | ["选项1", "选项2"] |
| 手机号码 | value以字符串格式传入 | 18858885666 |
| 日期选择 | 以自定义字段配置为准, value以yyyy-mm-dd hh:mm格式传入 | 2023-08-01 12:30 |

| 邮箱 | value以字符串格式传入 | 123456@163.com |

| 层级选择 | 选项层级、选项值，以及和选项路径所有节点的选项层级、选项值。选项层级以数字传入，选项值为字符串 | [{"level": "1", "value": "选项1"}, {"level": "2", "value:": "选项1-1"}] |
| 关联字段 | value以字符串格式传入 | 关联字段值 |
| 表格 | 每一行根据序号值区分，每一行中，每个表格值将titleCode和value 传入 Json格式字符串 | [{\"sequence\": 1, \"columnValues\": [{\"titleCode\": \"NAME\", \"value\": \"张三\"}, {\"titleCode\": \"AGE\", \"value\": \"26\"}]},<br><br>{\"sequence\": 2, \"columnValues\": [{\"titleCode\": \"NAME\", \"value\": \"李四\"}, {\"titleCode\": \"AGE\", \"value\": \"28\"}]}] |
| 人员 | value 以花名册编号传入 Json格式字符串 | ["EMP_NUMBER_01", "EMP_NUMBER_02"] |
| 单据部门 | 组织编码以字符串形式传入 | 00000001 |
| 单据项目 | value以项目编码传入 Json格式字符串 | ["PROJECT_1", "PROJECT_2"] |
| 关联单据 | 费控系统单据编号，以字符串形式传入 | 2023080202288221 |
| 申请金额 | 小数位和整数位以自定义字段配置为准，value以字符串格式传入 | 66.6 |
| 城市 | 城市编码，以字符串形式传入 | 20 |
| 供应商 | value以供应商编号传入 | SUP20211109091900 |
| 客户 | value以客户编号传入 Json格式字符串 | ["CUSTOMER_01", "CUSTOMER_02"] |
| 合同 | value以合同编号传入 | 8768768768768 |
| 成本中心 | value以成本中心编号传入 | center001 |
| 档案选择 | value以自定义档案选项编码传入 | option01 |
| 部门 | value以薪福通组织编码传入 Json格式字符串 | ["ORG_01", "ORG_02"] |
| 项目 | value以项目编码传入 Json格式字符串 | ["PROJECT_1", "PROJECT_2"] |
| 事业部 | value以薪福通组织编码传入 | 00000211 |
| 事项 | 配置为必填时，业务类型编码必传，且下级存在事项和事由时，事项编码和事由编码必传。value以“业务类型编码,事项编码,事由编码”传入 Json格式字符串 | [\"MATTER_TYPE_1\", \"MATTER_MISSION_1\", \"MATTER_DETAIL_1\"] |
| 收款账户 | value以个人或供应商银行账户号，开户行，户名传入 Json格式字符串；匹配启用状态的个人或供应商收款账户 | [\"62256255478532541\", \"中国银行\", \"小朱\"] |

 
## 调用方式 
 METHOD: POST  
 CONTENT-TYPE: application/json 
## 接口输入参数 
 
| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| header | OBJECT |  | 请求头根节点 |  |

| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| body | OBJECT |  | 请求体根节点 |  |
| body.eventNumber | STRING | 是 | 单据表单编号 | 01230705110652000636 |
| body.outRelId | STRING | 否 | 外部单据编号 |  |
| body.billCompanyCode | STRING | 否 | 薪福通法人公司编码，不传默认为申请人所属公司；<br><br>传入的单据公司与申请人不一致时，单据部门编号必传 | Fx60001 |
| body.billDepartmentCode | STRING | 否 | 薪福通部门组织编码，不传默认为申请人所属部门；<br><br>传入的单据部门与申请人不一致时，单据公司编号必传 | 00000003 |
| body.empNumber | STRING | 是 | 申请人薪福通花名册员工号 |  |
| body.billStatus | STRING | 是 | DRAFT：草稿；APPRV：审批通过；APPRV_PEND：审批中 | APPRV |
| body.customFieldValues | ARRAY | 否 | 自定义字段值列表 |  |
| body.customFieldValues.fieldNumber | STRING | 是 | 自定义字段编码 | MULTI_INPUT_170948711 |
| body.customFieldValues.fieldValue | STRING | 否 | 请参考接口描述中自定义字段传值格式和说明 | 多行文本 |
| body.multiPersonCollections | ARRAY | 否 | 多人收款信息列表 |  |
| body.multiPersonCollections.empNumber | STRING | 是 | 薪福通花名册员工号 | 88888888 |
| body.multiPersonCollections.depoName | STRING | 是 | 开户名 | 孔明 |
| body.multiPersonCollections.depoBankName | STRING | 是 | 开户行名称 | 中国银行 |
| body.multiPersonCollections.depoBranBankName | STRING | 是 | 开户支行名称 | 中国银行杭州分行 |
| body.multiPersonCollections.accountNumber | STRING | 是 | 银行账户 | 6666666666666666666 |
| body.multiPersonCollections.payAmount | LONG | 是 | 最多9位整数，2位小数 | 18 |
| body.relatedBillId | LONG | 否 | 费控系统内关联申请单的单据编号 |  |
| body.billApplyAuthCheckType | STRING |  | Y-校验单据发起权限<br>N-不校验单据发起权限<br>不传，默认为Y-校验单据发起权限 | Y |
