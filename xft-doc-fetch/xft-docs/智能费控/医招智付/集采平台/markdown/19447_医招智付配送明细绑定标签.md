# 医招智付配送明细绑定标签

## 接口调用地址 
 生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/purchase-settle/auto-inventory/distribution-detail/bind-tag 
 测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/purchase-settle/auto-inventory/distribution-detail/bind-tag 
 ## 接口描述 
**接口功能：**
医招智付-集采平台-自动入库模块，用于给药品配送明细绑定自定义标签，比如：临期药品、散装药品等；
**接口设计：**
* 批量接口，配送明细最大不超过100条，允许部分成功，部分失败。
* 每条配送明细可以绑定多个标签，最大不超过10个。
* 一次报文请求中配送明细不可重复。 
## 调用方式 
 METHOD: POST  
 CONTENT-TYPE: application/json

## 接口输出参数
 
| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| root | OBJECT |  | 响应头根节点 |  |

| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| root | OBJECT |  | 响应体根节点 |  |
| root.body | ARRAY | 否 | 只返回错误数据列表 |  |
| root.body.distributionCode | STRING | 否 | 配送明细编码 | "S2025112700001" |

## 接口调用示例 
 ### 请求报文
```json
[
    {
        "distributionCode": "S2025112700001",
        "tagInfoList": [
            "000001",
            "000002"
        ]
    }
]
```
### 响应报文
全部成功
```json
{
    "body": [],
    "returnCode": "SUC0000",
    "success": true
}
```

部分成功
```json
{
    "body": [
        {
            "distributeCode": "S13520251217000032",
            "errorMessage": "配送明细不存在;"
        },
        {
            "distributeCode": "S1352025121700002",
            "errorMessage": "标签25039216020不存在或未启用;"
        }
    ],
    "returnCode": "SUC0000",
    "success": true
}
```

全部失败
```json
{
    "body": [
        {
            "distributeCode": "S13520251217000032",
            "errorMessage": "配送明细不存在;"
        }
    ],
    "returnCode": "E0002",
    "success": false
}
```
