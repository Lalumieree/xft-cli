# 医招智付供应商端配送明细导入

## 接口调用地址 
 生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/purchase-settle/supplier/distribution-detail/import 
 测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/purchase-settle/supplier/distribution-detail/import 
 ## 接口描述 
药品配送明细导入，将集采平台业务系统中的药品配送明细数据推送至医招智付（供应商端）应用。

以药品配送明细编码作为唯一键，重复导入时进行更新操作。

注意：
1. 批量接口，允许部分成功；
2. 单次导入上限为1000；
3. 已从供应商端推送至医院端的数据，不支持更新。
 
## 调用方式 
 METHOD: POST  
 CONTENT-TYPE: application/json

## 错误码说明 
| 错误码| 错误信息|解决方案|
  | --- | --- | --- |
  | E0002| 导入的配送明细不能为空| 输入需要导入的配送明细 |
 | E0002| 医院薪福通企业号不存在| 输入正确的医院薪福通企业号 |
  | E0002| 导入医院的薪福通企业号不能为空| 输入导入配送明细对应的医院薪福通企业号 |
  | E0002| 配送明细编码不能为空| 输入每条导入配送明细对应的配送明细编码 |
  | EROOR1| 系统错误| 稍后再试，或联系开发人员 |
  | EROOR2| 导入数据超过最大限制1000条|调整导入数据至1000以下 |
  | EROOR2| 暂未与该医院建立绑定关系，请联系医院绑定|联系医院建立绑定关系 |
  | EROOR2|以下配送明细编码重复：xxxx|检查导入数据，修改导入的配送明细编码|
