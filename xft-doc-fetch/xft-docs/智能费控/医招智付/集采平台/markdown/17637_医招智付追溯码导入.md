# 医招智付追溯码导入

## 接口调用地址 
 生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/purchase-settle/trace-code/import 
 测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/purchase-settle/trace-code/import 
 ## 接口描述 
将追溯码导入医招智付-追溯码台账中，医招智付会调取阿里码上放心接口解析追溯码信息。

注意：
1. 单次最大导入上限为80条；
2. 导入的追溯码长度只能为20位；
3. 导入前，请先在医招智付中配置阿里码上放心授权信息。 
## 调用方式 
 METHOD: POST  
 CONTENT-TYPE: application/json

## 接口调用示例 
 ### 请求报文
```json
{  
    "deliverEnterprise": "泉州鹭燕医药有限公司",  
    "traceCodeList": [  
        "84004560440850000177"  
    ]  
}  
```
### 响应报文
```json
{  
    "body":"追溯码导入成功",  
    "returnCode":"SUC0000",  
    "success":true",  
    "message":"成功"  
}  
```  
## 错误码说明 
| 错误码| 错误信息|解决方案|
  | --- | --- | --- |
  | E0002| 配送企业名称不能为空| 传入配送企业名称|
  | E0002| 输入正确格式的20位追溯码| 输入正确的追溯码|
  | E0002| 追溯码导入上限为80条| 调整单次导入追溯码数量|
  | E0002| 配送企业名称不可超过200个字符| 输入正确的配送企业名称|
  | EROOR1| 系统错误| 稍后再试，或联系开发人员|
  | EROOR2| 授权码为空，请在医招智付中维护授权信息| 在“医招智付-集采平台-规则设置-授权码”中维护正确的阿里授权信息|
  | EROOR2| 未找到对应码上放心供应商，请精准维护码上放心供应商全称| 输入的配送企业名称不正确或该企业尚未在阿里码上放心平台注册|
