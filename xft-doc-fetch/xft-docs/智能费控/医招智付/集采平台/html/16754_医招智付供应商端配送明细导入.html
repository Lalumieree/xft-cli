医招智付供应商端配送明细导入

接口调用地址

生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/purchase-settle/supplier/distribution-detail/import

测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/purchase-settle/supplier/distribution-detail/import

接口描述

药品配送明细导入，将集采平台业务系统中的药品配送明细数据推送至医招智付（供应商端）应用。

以药品配送明细编码作为唯一键，重复导入时进行更新操作。

注意：

1. 批量接口，允许部分成功；

2. 单次导入上限为1000；

3. 已从供应商端推送至医院端的数据，不支持更新。

调用方式

METHOD: POST

CONTENT-TYPE: application/json

接口输入参数

参数类型必填说明示例
rootOBJECT请求头根节点
参数类型必填说明示例
rootOBJECT请求体根节点
root.hospitalXftCodeSTRING是导入医院的薪福通企业号"XFT00001"
root.distributionDetailsARRAY是导入的配送明细数据，上限为1000
root.distributionDetails.distributionCodeSTRING是唯一标识"S2025072922215"
root.distributionDetails.orderCodeSTRING否订单编码"O202507111"
root.distributionDetails.productNameSTRING否产品名称"布洛芬"
root.distributionDetails.productCodeSTRING否产品编码"PROD202507777"
root.distributionDetails.projectNameSTRING否项目名称"联合限价阳光采购"
root.distributionDetails.tradeNameSTRING否商品名称"通用止痛药"
root.distributionDetails.hospListCodeSTRING否院内目录编码"CSBESTGAME20250729"
root.distributionDetails.productSpecSTRING否产品规格"20ml:10mg"
root.distributionDetails.productModSTRING否产品剂型"注射剂"
root.distributionDetails.productPackageSTRING否产品包装"100ml:10g(精制鱼油):1.2g(卵磷脂)×1瓶/瓶"
root.distributionDetails.approveNoSTRING否批准文号"国药准字J20150040"
root.distributionDetails.batchNumberSTRING否批次号"100-3333"
root.distributionDetails.pacConvRatSTRING否转换比"1.0/1"
root.distributionDetails.purchaseCountBIGDECIMAL否整数位不能超过15位，小数位不能超过6位200.5
root.distributionDetails.deliveryCountBIGDECIMAL否整数位不能超过15位，小数位不能超过6位12.4
root.distributionDetails.deliveryAmountBIGDECIMAL否整数位不能超过15位，小数位不能超过6位1888.5
root.distributionDetails.deliveryTimeSTRING否yyyy-MM-dd HH:mm:ss"2024-07-10 15:24:00"
root.distributionDetails.deliverEnterpriseNameSTRING否配送企业名称"南京新时代药业有限公司"
root.distributionDetails.produceEnterpriseNameSTRING否生产企业名称"上饶新时代建设有限公司"
root.distributionDetails.purchasePriceBIGDECIMAL否整数位不能超过15位，小数位不能超过6位18.5
root.distributionDetails.highSelLmtPriceBIGDECIMAL否整数位不能超过15位，小数位不能超过6位20.87
root.distributionDetails.expiredDateSTRING否yyyy-MM-dd"2024-12-27"
root.distributionDetails.receiveAddressSTRING否收货地址"江西省上饶市重阳大道999号"
root.distributionDetails.storageRoomSTRING否库房"一号库房"
root.distributionDetails.contactNameSTRING否联系人"TOM"
root.distributionDetails.selInvoiceTwoSTRING否发票号码（二票）"202507290000"
root.distributionDetails.selInvoiceDateTwoSTRING否yyyy-MM-dd HH:mm:ss"2025-08-28 19:00:00"
root.distributionDetails.selInvoiceOneSTRING否发票号码（一票）"12345678901234657890"
root.distributionDetails.selInvoiceDateOneSTRING否yyyy-MM-dd HH:mm:ss"2025-08-28 19:00:00"
root.distributionDetails.traceCodesSTRING否多个追溯码使用英文逗号,拼接"123354645656,434354545657"
参数类型必填说明示例
rootOBJECTquery根节点

接口输出参数

参数类型必填说明示例
rootOBJECT响应头根节点
参数类型必填说明示例
rootOBJECT响应体根节点
root.returnCodeSTRING是调用失败：EROOR1；
执行失败：EROOR2；
执行成功：SUC0000；
参数错误：E0002		"SUC0000"
root.successBOOLEAN是成功时 返回true；
失败时 返回false		true
root.messageSTRING是调用成功时，返回“成功”；
调用失败时，返回错误消息。		"成功"
root.bodyOBJECT是导入结果
root.body.succeedSizeINTEGER是导入成功条数100
root.body.failedSizeINTEGER是导入失败条数0
root.body.resultBOOLEAN是导入部分成功、全部成功：true；
全部失败：false		true
root.body.errorListARRAY导入失败的数据
root.body.errorList.distributeCodeSTRING配送明细编码"S123456789"
root.body.errorList.errorMessageSTRING错误信息"配送企业名称长度不能超过200位"

错误码说明

错误码错误信息解决方案
E0002导入的配送明细不能为空输入需要导入的配送明细
E0002医院薪福通企业号不存在输入正确的医院薪福通企业号
E0002导入医院的薪福通企业号不能为空输入导入配送明细对应的医院薪福通企业号

EROOR1系统错误稍后再试，或联系开发人员
