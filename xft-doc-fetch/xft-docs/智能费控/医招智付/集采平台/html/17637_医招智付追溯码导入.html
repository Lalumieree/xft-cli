医招智付追溯码导入

接口调用地址

生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/purchase-settle/trace-code/import

测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/purchase-settle/trace-code/import

接口描述

将追溯码导入医招智付-追溯码台账中，医招智付会调取阿里码上放心接口解析追溯码信息。

注意：

1. 单次最大导入上限为80条；

2. 导入的追溯码长度只能为20位；

3. 导入前，请先在医招智付中配置阿里码上放心授权信息。

调用方式

METHOD: POST

CONTENT-TYPE: application/json

接口输入参数

参数类型必填说明示例
rootOBJECT请求头根节点
参数类型必填说明示例
rootOBJECT请求体根节点
root.deliverEnterpriseSTRING是查询公司在阿里码上放心平台注册的ID，需要与阿里码上放心平台注册的公司名称一致"南京华北医药有限责任公司"
root.traceCodeListARRAY否单个追溯码长度为20；
单次导入上限为80；
导入内容为空，直接返回“成功”。		["84004560440850000177"]
参数类型必填说明示例
rootOBJECTquery根节点

接口输出参数

参数类型必填说明示例
rootOBJECT响应头根节点
参数类型必填说明示例
rootOBJECT响应体根节点
root.successBOOLEAN是成功：true；
失败：false		true
root.messageSTRING是执行成功时，会返回的信息：”成功“；
执行失败时，会返回错误信息。		"成功"
root.returnCodeSTRING是调用失败：EROOR1；
执行失败：EROOR2；
执行成功：SUC0000；
参数错误：E0002		"SUC0000"
root.bodySTRING是执行成功时，返回"追溯码导入成功""追溯码导入成功"

接口调用示例

请求报文


{  
    "deliverEnterprise": "泉州鹭燕医药有限公司",  
    "traceCodeList": [  
        "84004560440850000177"  
    ]  
}

响应报文


{  
    "body":"追溯码导入成功",  
    "returnCode":"SUC0000",  
    "success":true",  
    "message":"成功"  
}

错误码说明

错误码错误信息解决方案
E0002配送企业名称不能为空传入配送企业名称
E0002输入正确格式的20位追溯码输入正确的追溯码
E0002追溯码导入上限为80条调整单次导入追溯码数量
E0002配送企业名称不可超过200个字符输入正确的配送企业名称
EROOR1系统错误稍后再试，或联系开发人员
EROOR2授权码为空，请在医招智付中维护授权信息在“医招智付-集采平台-规则设置-授权码”中维护正确的阿里授权信息
EROOR2未找到对应码上放心供应商，请精准维护码上放心供应商全称输入的配送企业名称不正确或该企业尚未在阿里码上放心平台注册
