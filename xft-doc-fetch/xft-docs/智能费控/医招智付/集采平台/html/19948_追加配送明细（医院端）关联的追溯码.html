追加配送明细（医院端）关联的追溯码

接口调用地址

生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/purchase-settle/distribution-detail/relate-trace-code/import

测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/purchase-settle/distribution-detail/relate-trace-code/import

接口描述

对医招智付-集采平台-自动入库里的配送明细追加追溯码。

注：

1. 是**追加**追溯码，并非覆盖；

2. 单次导入追溯码****上限2000条****；

3. 导入重复的追溯码时，会去重处理；

4. 如需覆盖追溯码，请使用 “导入集采平台药品配送明细”api。

调用方式

METHOD: POST

CONTENT-TYPE: application/json

接口输入参数

参数类型必填说明示例
rootOBJECT请求头根节点
参数类型必填说明示例
rootOBJECT是请求体根节点
root.distributionCodeSTRING是配送明细唯一编码"S2026012901"
root.traceCodeListARRAY是上限为2000条追溯码；
单条追溯码长度上限为100。		["11111111111111111111111111"]
参数类型必填说明示例
rootOBJECTquery根节点

接口输出参数

参数类型必填说明示例
rootOBJECT响应头根节点
参数类型必填说明示例
rootOBJECT是响应体根节点
root.returnCodeSTRING是调用失败：EROOR1；
执行失败：EROOR2；
执行成功：SUC0000；
参数错误：E0002		"SUC0000"
root.successBOOLEAN是成功时 返回true；
失败时 返回false		true
root.messageSTRING是调用成功时，返回“成功”；
调用失败时，返回错误消息。		"成功"
root.bodyOBJECT是关联追溯码结果
root.body.associationQuantityINTEGER是返回配送明细已关联的追溯码数量，如，初始关联追溯码0条：
1. 第一次导入2000条，返回2000；
2. 第二次导入2000条，则返回4000；
3. 依此类推...		2000

接口调用示例

请求报文


{
    "distributionCode": "S2026012901",
    "traceCodeList": [
        "12345678901234567890",
        "111111111111111111111111111111"
    ]
}

响应报文


{
    "body": {
        "associationQuantity": 2
    },
    "returnCode": "SUC0000",
    "success": true,
    "message": "成功"
}
