医招智付配送明细绑定标签

接口调用地址

生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/purchase-settle/auto-inventory/distribution-detail/bind-tag

测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/purchase-settle/auto-inventory/distribution-detail/bind-tag

接口描述

**接口功能：**

医招智付-集采平台-自动入库模块，用于给药品配送明细绑定自定义标签，比如：临期药品、散装药品等；

**接口设计：**

* 批量接口，配送明细最大不超过100条，允许部分成功，部分失败。

* 每条配送明细可以绑定多个标签，最大不超过10个。

* 一次报文请求中配送明细不可重复。

调用方式

METHOD: POST

CONTENT-TYPE: application/json

接口输入参数

参数类型必填说明示例
rootOBJECT请求头根节点
参数类型必填说明示例
rootARRAY请求体根节点
root.distributionCodeSTRING是药品配送明细编码"S2025112700001"
root.tagInfoListARRAY是标签编码列表，每个标签编码代表一种标签，比如"临期药品"等
标签编码长度最长32位。
标签编码查询路径: 医招智付-集采平台-规则设置-匹配规则-自动入库-自定义标签		["000001","00002"]
参数类型必填说明示例
rootOBJECTquery根节点

接口输出参数

参数类型必填说明示例
rootOBJECT响应头根节点
参数类型必填说明示例
rootOBJECT响应体根节点
root.bodyARRAY否只返回错误数据列表
root.body.distributionCodeSTRING否配送明细编码"S2025112700001"
root.body.errorMessageSTRING否错误信息"标签编码不存在"
root.returnCodeSTRING否错误码：
SUC0000 - 成功；部分成功也算成功
E0002 - 业务错误，具体报错参考errorMessage字段		"SUC0000"
root.successBOOLEAN否接口执行结果
true - 执行成功；部分成功也返回true
false - 全部失败		true
