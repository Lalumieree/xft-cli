发票签收

接口调用地址

生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/invoice/updateVerifySts

测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/invoice/updateVerifySts

接口描述

将发票签收状态设置为已签收，并改变关联单据的核验状态。

调用方式

METHOD: POST

CONTENT-TYPE: application/json

接口输入参数

参数类型必填说明示例
headerOBJECT请求头根节点
参数类型必填说明示例
bodyOBJECT是请求体根节点
body.billTypeSTRING是取值参考[发票类型枚举](https://xft.cmbchina.com/open/#/doc/open-document?id=10692&mid=11585)4
body.signStaffNameSTRING是发票签收人名称test1
body.signStaffIdSTRING否含义与empId相同。signStaffId、empNumber和empPhone必填其一U0000
body.empNumberSTRING否signStaffId、empNumber和empPhone必填其一。41904
body.empPhoneSTRING否signStaffId、empNumber和empPhone必填其一。14900000000
body.invoiceInfoOBJECT发票类型不是为5,6,7,E,F,Z时必填
body.invoiceInfo.invoiceNoSTRING发票类型为1，2，3，4，8，9，A,B,C,D,G,H时有值04829364
body.invoiceInfo.invoiceCodeSTRING发票类型为1，2，3，4，8，9，A,B,C,D,G,H时有值。5000204130
body.invoiceInfo.trafficCodeSTRING发票类型为7,E,F时有值5000204130
body.invoiceInfo.trafficNoSTRING发票类型为7,E,F时为票据号码；发票类型为5，6时为票号04829364
body.trafficInfoOBJECT发票类型为5,6,7,E,F,Z时必填
body.trafficInfo.invoiceNoSTRING发票类型为1，2，3，4，8，9，A,B,C,D,G,H时有值04829364
body.trafficInfo.invoiceCodeSTRING发票类型为1，2，3，4，8，9，A,B,C,D,G,H时有值。5000204130
body.trafficInfo.trafficCodeSTRING发票类型为7,E,F时有值5000204130
body.trafficInfo.trafficNoSTRING发票类型为7,E,F时为票据号码；发票类型为5，6时为票号04829364
参数类型必填说明示例
queryOBJECTquery根节点

接口输出参数

参数类型必填说明示例
headerOBJECT响应头根节点
参数类型必填说明示例
bodyOBJECT响应体根节点
body.successBOOLEANtrue-成功,false-失败true
body.messageSTRING成功或错误信息调用成功

接口调用示例

**请求示例**


{
    "billType": "4",
    "signStaffName": "test1",
    "signStaffId": "U0000",
    "empNumber": "41904",
    "empPhone": "14900000000",
    "invoiceInfo": {
        "invoiceNo": "04829364",
        "invoiceCode": "5000204130",
        "trafficNo": null,
        "trafficCode": null
    },
    "trafficInfo": {
        "invoiceNo": null,
        "invoiceCode": null,
        "trafficNo": null,
        "trafficCode": null
    }
}

**返回示例**

正常：


{
    "success": true,
    "message": "调用成功"
}

异常：

如果没有传入必填信息，返回样例：

未传入发票类型


{
    "success": false,
    "message": "发票类型必填，请检查信息是否完整"
}

未传入签收人姓名


{
    "success": false,
    "message": "签收人姓名必填，请检查信息是否完整"
}

如果没有传入签收人信息，返回样例：


{
    "success": false,
    "message": "缺少签收人信息"
}

如果找不到对应员工，返回样例：


{
    "success": false,
    "message": "找不到对应的员工信息:U0000"
}

如果传入签收人信息与签收人姓名不一致，返回样例：


{
    "success": false,
    "message": "发票签收人姓名与发票签收人信息不一致"
}

如果没有传发票信息，返回样例：


{
    "success": false,
    "message": "发票信息为空或不完整"
}

如果传入了其他公司的发票信息，返回样例：


{
    "success": false,
    "message": "存在其他公司的发票"
}

如果发票完全不属于当前公司，返回样例：


{
    "success": false,
    "message": "发票不在当前公司下"
}

错误码说明
