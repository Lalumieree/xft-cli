# 单据签收

## 接口调用地址 
 生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/bills/signBills 
 测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/bills/signBills 
 ## 接口描述 
批量签收单据，将单据下发票状态全部设置为已签收但不覆盖之前已签收发票的签收人信息。 
## 调用方式 
 METHOD: POST  
 CONTENT-TYPE: application/json 
## 接口输入参数 
 
| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| header | OBJECT |  | 请求头根节点 |  |

| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| body | OBJECT | 是 | 请求体根节点 |  |
| body.billIdList | ARRAY | 是 | 建议一次签收不要超过100个单据 | [2022121901730247, 2023011801809008] |
| body.empNam | STRING | 是 | 签收人姓名 | 张三 |
| body.empId | STRING | 否 | 员工ID，为签收人信息。员工ID、员工编号和员工手机号至少填一项 | U0000 |
| body.empNumber | STRING | 否 | 员工编号，为签收人信息。员工ID、员工编号和员工手机号至少填一项 | E0064 |
| body.empPhone | STRING | 否 | 员工手机号，为签收人信息。员工ID、员工编号和员工手机号至少填一项 | 11111111111 |

| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| query | OBJECT |  | query根节点 |  |
 
## 接口输出参数
 
| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| header | OBJECT |  | 响应头根节点 |  |

| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| body | OBJECT |  | 响应体根节点 |  |
| body.success | BOOLEAN |  | 请求结果 | true |
| body.message | STRING |  | 返回信息 | 调用成功 |
 
## 接口调用示例 
 {BaseURL}/api/itrip-gateway/v1/bills/signBills
### **请求示例**
正常：
```json
{
    "billIdList": [
        2022121901730247,
        2023011801809008
    ],
    "empNam": "张三",
    "empId": "U0000",
    "empNumber": null,
    "empPhone": 11111111111,
}
```
### **返回示例**
如果没有传入签收人姓名，返回样例：
```json
{
    "success": false,
    "message": "请传入签收人姓名"
}
```
如果没有传入签收人信息，返回样例：
```json
{
    "success": false,
    "message": "缺少签收人信息"
}
```
如果找不到对应员工，返回样例：
```json
{
    "success": false,
    "message": "找不到对应的员工信息:U0000"
}
```
如果签收人姓名和签收人信息不一致，返回样例：
```json
{
    "success": false,
    "message": "单据签收人姓名与单据签收人信息不一致"
}
```
如果没有传单据信息，返回样例：
```json
{
    "success": false,
    "message": "单据信息为空"
}
```
如果传了其他公司的单据，返回样例：
```json
{
    "success": false,
    "message": "存在其他公司的单据"
}
```
如果单据完全不属于当前公司，返回样例：
```json
{
    "success": false,
    "message": "单据不在当前公司下"
}
```
