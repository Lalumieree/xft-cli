批量新增供应商

接口调用地址

生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/supplier/batch-add

测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/supplier/batch-add

接口描述

批量新增供应商，单次最多新增50个供应商，不支持部分成功

调用方式

METHOD: POST

CONTENT-TYPE: application/json

接口输入参数

参数类型必填说明示例
rootOBJECT请求头根节点
参数类型必填说明示例
rootARRAY请求体根节点
root.availableOrgCodeListARRAY否数据来源：智能费控-档案设置-档案导出-导出法人公司
可见法人公司编码列表
若availableOrgIdList列表为空，则使用本Code列表对应的公司作为可见法人公司列表		["Fx00000"]
root.availableOrgIdListARRAY数据来源：智能费控-档案设置-档案导出-导出法人公司
可见法人公司id列表
优先使用id对应的可见法人公司列表		["CORx80000"]
root.availableOrgTypeSTRING是1-部分公司 0-全部公司1
root.corOrgNbrSTRING否统一信用代码125444444444450005
root.customFieldValuesARRAY自定义字段需要填入的值
root.customFieldValues.customFieldCodeSTRING智能费控-档案设置=字段设置-供应商字段编码SELECT_140332273
root.customFieldValues.customFieldValueSTRING自定义字段是单值情况下填入，如果是多值，无需填入
root.customFieldValues.multiValuesARRAY自定义字段是多值情况下填入，如果是单值，无需填入
root.customFieldValues.multiValues.idSTRING与自定义多选选项名称一致选项1
root.customFieldValues.multiValues.nameSTRING自定义多选选项名称选项1
root.enableDateSTRING否格式：yyyy-MM-dd
默认当天		2024-10-21
root.recAccListARRAY收款账户集合
root.recAccList.accountModeSTRING账户类型: BANK-银行账户 DIGITAL_YUAN-数字人民币 默认银行账户BANK
root.recAccList.acctNoSTRING银行账号不能为空 银行账号需是不能超过30位的字母或数字6217847384349543545
root.recAccList.acctTypeSTRING1-对公账户 0-个人账户1
root.recAccList.collectingCompanyNameSTRING数字人民币账户字段，银行账户无需处理测试中心
root.recAccList.depoBankSTRING银行支行联行号，可通过公共接口中的银行接口获取，账户类型为银行账户必填105399100293
root.recAccList.depoNameSTRING是银行户名不能为空
账户类型为数币时此字段代表钱包名称		测试账户
root.recAccList.mainAcctSTRING是1-是 0-否1
root.recAccList.remarkSTRING备注测试新增
root.recAccList.stateSTRING是1-是 0-否1
root.recAccList.wthrDigitalYuanMainAcctSTRING数字人民币账户字段，银行账户无需处理
1-是，0-否		0
root.supIdSTRING供应商编号必须是100位以内的字母或数字，不可重复，不传则由系统生成SUP20241028164158777
root.supNamSTRING是供应商名称czc新增测试11
root.supStsSTRING是供应商状态
1-启用 0-停用		1
root.supTypeIdSTRING是智能费控-档案设置-字段设置-供应商-供应商类型0
参数类型必填说明示例
rootOBJECTquery根节点

接口输出参数

参数类型必填说明示例
rootOBJECT响应头根节点
参数类型必填说明示例
rootOBJECT响应体根节点
root.bodySTRING否返回体批量新增供应商成功！
root.returnCodeSTRING否错误码SUC0000
root.successBOOLEAN否是否调用成功true
root.messageSTRING否错误提示成功
