编辑合同档案

接口调用地址

生产环境地址：https://api.cmbchina.com/itrip/xft-api/v2/archives/contract/edit

测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v2/archives/contract/edit

接口描述

在公司下编辑合同档案，支持编辑付款合同、框架合同，支持传入合同基本信息、合同适用范围、合同附件、付款合同付款条款、合同受益期、合同自定义字段等信息。

调用方式

METHOD: POST

CONTENT-TYPE: application/json

接口输入参数

参数类型必填说明示例
rootOBJECT请求头根节点
参数类型必填说明示例
bodyOBJECT是请求体根节点
body.contractCompanyCodeSTRING是合同公司编码SH01
body.billContractSeqIdLONG否合同唯一自增ID1952
body.contractDepartmentCodeSTRING是责任部门编码0000
body.contractEmpNumberSTRING是员工编号00210416
body.applyAllUserBOOLEAN是合同是否适用全部员工false
body.contractGroupsARRAY否合同适用范围为全部时，不支持配置合同适用范围
body.contractGroups.typeSTRING是EMPLOYEE 员工
DEPARTMENT 部门
POST 岗位
JOB 职位
ROLE 角色
RANK 职级		EMPLOYEE
body.contractGroups.typeIdNameSTRING是除类型为角色（type: ROLE）需要传角色名称，其余情况传值为对应类型的编码。eg：员工编码（type: EMPLOYEE）00210416
body.contractTypeCodeSTRING是合同类型的编码信息CONTRATTYPE20230109192232
body.contractNameSTRING是合同名称测试API新增付款合同2
body.supplierCodeSTRING是供应商编码SUP20240304175207
body.contractStartDateSTRING是日期格式：yyyy-MM-dd2024-05-01
body.contractEndDateSTRING是日期格式：yyyy-MM-dd2026-05-01
body.currencyCodeSTRING是根据公司开启的金额类型，eg：CNY、AUD.......CNY
body.contractAmtBIGDECIMAL是金额精度：(17,2)0
body.contractNbrSTRING是合同编码CONTRAT2024010920
body.startPeriodSTRING否初始周期2024-05
body.contractAttachmentsARRAY否1，合同附件可为空
2，传入合同附件时，合同附件数量不超过十个
3，传入合同附件时，合同附件的唯一ID和合同附件的UUID名称均不能为空
body.contractAttachments.seqIdLONG是合同附件的唯一ID33226
body.contractAttachments.fileNamSTRING是合同附件的UUID名称cd7bbf17-8971-404c-9558-bc837abda065
body.remarkSTRING否合同备注付款合同
body.payProvisionListARRAY否1，合同类型为付款合同，付款条款信息必填
2，合同类型为归档合同，不支持添加付款条款信息
3，合同类型为框架合同，不支持添加付款条款信息
body.payProvisionList.payStageNamSTRING否付款阶段名称阶段一
body.payProvisionList.payAmtBIGDECIMAL否金额精度：(17,2)100
body.payProvisionList.depoNameSTRING否付款账户的开户名信息滨江小蜜蜂公司dx
body.payProvisionList.acctNoSTRING否付款账户的账号22154545000000
body.payProvisionList.payDateSTRING否日期格式：yyyy-MM-dd2024-05-14
body.payProvisionList.provisionSTRING否付款条款信息付款阶段一条款信息
body.payProvisionList.remarkSTRING否付款条款的备注信息付款阶段备注信息
body.payProvisionList.payProvisionSeqIdSTRING付款条款的唯一ID，编辑付款条款的时候必传，在已有合同内容新增付款条款的时候不传1201
body.payProvisionList.initialReimbursedAmountBIGDECIMAL金额精度：(17,2)
不能大于条款金额，作为条款的初始已经报销的金额，用于导入已经使用过的合同		0
body.benefitPeriodListARRAY否传入合同受益期信息
body.benefitPeriodList.periodLONG否受益期的阶段1
body.benefitPeriodList.useFlagLONG否受益期是否停用：
1 停用
0 启用		1
body.customFieldValuesARRAY自定义字段值
body.customFieldValues.customFieldCodeSTRING是必填，自定义字段唯一编码INFINITY_SELECT_170255122
body.customFieldValues.customFieldValueSTRING数字，日期，单选，单行文本，多行文本等自定义字段类型值填写位置2523456346564564534汉字
body.customFieldValues.customFieldNameSTRING非必填，可通过自定义字段编码获取自定义字段名称下拉选择多选
body.customFieldValues.multiValuesARRAY下拉选择，多选等自定义字段类型多选时的值填写位置{ "id": "选项3", "name": "选项3" }
body.customFieldValues.multiValues.idSTRING否多选的值 value选项3
body.customFieldValues.multiValues.nameSTRING是多选的显示 key选项3
参数类型必填说明示例
rootOBJECTquery根节点
root.EMPNUMBERSTRING操作人编码00210416

接口输出参数

参数类型必填说明示例
rootOBJECT响应头根节点
参数类型必填说明示例
bodyOBJECT响应体根节点
body.bodyBOOLEAN否是否编辑成功true
body.returnCodeSTRING否返回编码SUC0000
body.successBOOLEAN否接口调用是否成功true
body.messageSTRING否明细信息成功
