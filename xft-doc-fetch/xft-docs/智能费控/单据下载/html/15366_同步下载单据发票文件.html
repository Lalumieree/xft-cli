同步下载单据发票文件

接口调用地址

生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/invoice/bill/download-invoice

测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/invoice/bill/download-invoice

接口描述

根据单据编号同步下载发票的全量文件（如果同一张发票有 pdf、ofd、xml、png的格式的文件则会下载4个文件），返回zip包；

如果单据关联发票文件过多(>50)则会返回指定的错误码(D0009),此时需分批下载-通过请求体中的 invoiceNumbers和invoiceCodes指定发票分批下载文件；

invoiceNumbers和invoiceCodes可以通过调用对应单据详情接口获取。

如《对公报账单》：开发文档 ->智能费控服务 -> 对公报账业务 -> 批量查询对公报账单详情；

《差旅报销单》：开发文档 ->智能费控服务 -> 报销单业务 -> 单笔查询差旅报销单详情；

ps：发票有invoiceNumber时传invoiceNumber ；如果没有invoiceNumber 时则传invoiceCode进行查询下载，invoiceNumbers 和 invoiceCodes 都传时，取两个条件的并集

调用方式

METHOD: POST

CONTENT-TYPE: application/json

接口输入参数

参数类型必填说明示例
rootOBJECT请求头根节点
参数类型必填说明示例
rootOBJECT请求体根节点
root.billIdLONG是单据编号2025032004147206
root.invoiceNumbersARRAY发票号集合 最多10个发票号[25519146042000000001,25519146042000000002]
root.invoiceCodesARRAY发票代码集合最多10个[234487841,234487842]
参数类型必填说明示例
rootOBJECTquery根节点

接口输出参数

参数类型必填说明示例
rootOBJECT响应头根节点
参数类型必填说明示例
rootOBJECT响应体根节点

错误码说明

错误码错误信息解决方案
D0003单据费用凭证不存在检查单据编号是否正确或是否关联费用凭证
D0009影像文件数量过大（>50）指定具体发票下载
