对账单生成

接口调用地址

生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/purchase-pay/reconciliation/core/service/generate

测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/purchase-pay/reconciliation/core/service/generate

接口描述

用于客户在采购付款的场景下，批量根据入库单生成对账单，并可选自动发起对账

- 单次最多支持500条入库单

- 入库单要求是同一个供应商和法人公司

- 入库单必须是需要对账的入库单，并且状态是待对账

- 如果选择了自动发起对账，则要求入库单的供应商和公司有绑定关系

调用方式

METHOD: POST

CONTENT-TYPE: application/json

接口输入参数

参数类型必填说明示例
rootOBJECT请求头根节点
参数类型必填说明示例
rootOBJECT请求体根节点
root.inveOrderIdsARRAY是最多500个["order1","order2"]
root.autoReconciliationBOOLEAN默认是falsetrue
root.remarkSTRING备注"备注123"
root.operatorEmpNumberSTRING是操作人员工号"8029148"
参数类型必填说明示例
rootOBJECTquery根节点

接口输出参数

参数类型必填说明示例
rootOBJECT响应头根节点
参数类型必填说明示例
rootOBJECT响应体根节点
root.returnCodeSTRING否错误码SUC0000
root.successBOOLEAN是是否成功false
root.messageSTRING否信息"请检查入库单是否为同一个供应商"
root.bodyOBJECT否返回体
root.body.billIdLONG是对账单号2025081204577220

接口调用示例

请求报文


{
    
    "inveOrderIds": [
        "2025-swq-2"
    ],
    "remark": "这是一个自动对账单",
    "operatorEmpNumber": "123456",
    "autoReconciliation": true
}

响应报文

- 示例1


{
    "body": 2025082804630478,
    "returnCode": "SUC0000",
    "success": true,
    "message": "成功"
}

- 示例2


{
    "returnCode": "EROOR2",
    "success": false,
    "message": "存在作废或重复发起对账的入库单：2025-swq-2"
}
