采购付款预入库单推送结果回写

接口调用地址

生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/purchase-pay/core/pre-inventory/push-result-update

测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/purchase-pay/core/pre-inventory/push-result-update

接口描述

预入库单推送结果回写接口，将每条预入库单的推送结果回写至采购付款。

注意：

单次回写上限为300条；

本接口对应图中红框里的流程。

![推送预入库单.png](https://s3gw.cmbchina.com/lw36.01-public-prd/public-a1dfd494-ade9-4302-b440-7f88e07b517b)

调用方式

METHOD: POST

CONTENT-TYPE: application/json

接口输入参数

参数类型必填说明示例
rootOBJECT请求头根节点
参数类型必填说明示例
rootOBJECT请求体根节点
root.pushResultListARRAY否最多支持300个，参数为空则不更新
root.pushResultList.inventoryCodeSTRING是预入库单编号"INV1234567"
root.pushResultList.resultBOOLEAN是成功：true，失败：falsetrue
root.pushResultList.msgSTRING推送成功时：填写推送成功的消息；
推送失败时：填写推送失败的消息。		"预入库单PRE000001不存在"
参数类型必填说明示例
rootOBJECTquery根节点

接口输出参数

参数类型必填说明示例
rootOBJECT响应头根节点
参数类型必填说明示例
rootOBJECT响应体根节点
root.successBOOLEAN是成功：true，失败：falsetrue
root.messageSTRING是执行成功时，会返回的信息：”成功“。
执行失败时，会返回错误信息。		"成功"
root.returnCodeSTRING是调用失败：EROOR1
执行失败：EROOR2
执行成功：SUC0000
参数错误：E0002		"SUC0000"
root.bodySTRING是执行成功时，返回“推送结果更新成功”"推送结果更新成功"

接口调用示例

请求报文


{
    "pushResultList": [
        {
            "inventoryCode": "20250829-2",
            "result": true,
            "msg": "His0002"
        },
        {
            "inventoryCode": "20250829-3",
            "result": false,
            "msg": "连接异常"
        }
    ]
}

响应报文


{
    "body": "推送结果更新成功",
    "returnCode": "SUC0000",
    "success": true,
    "message": "成功"
}

错误码说明

错误码错误信息解决方案
EROOR1系统错误稍后再试，或联系开发人员
E0002更新预入库单数量不得超过300条请确保单次更新的预入库单号数量少于300条
