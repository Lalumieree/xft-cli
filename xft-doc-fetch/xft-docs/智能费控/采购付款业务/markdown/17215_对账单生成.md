# 对账单生成

## 接口调用地址 
 生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/purchase-pay/reconciliation/core/service/generate 
 测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/purchase-pay/reconciliation/core/service/generate 
 ## 接口描述 
用于客户在采购付款的场景下，批量根据入库单生成对账单，并可选自动发起对账

- 单次最多支持500条入库单
- 入库单要求是同一个供应商和法人公司
- 入库单必须是需要对账的入库单，并且状态是待对账
- 如果选择了自动发起对账，则要求入库单的供应商和公司有绑定关系 
## 调用方式 
 METHOD: POST  
 CONTENT-TYPE: application/json

## 接口调用示例 
 ### 请求报文
```json
{
    
    "inveOrderIds": [
        "2025-swq-2"
    ],
    "remark": "这是一个自动对账单",
    "operatorEmpNumber": "123456",
    "autoReconciliation": true
}
```
### 响应报文
- 示例1
```json
{
    "body": 2025082804630478,
    "returnCode": "SUC0000",
    "success": true,
    "message": "成功"
}
```
- 示例2
