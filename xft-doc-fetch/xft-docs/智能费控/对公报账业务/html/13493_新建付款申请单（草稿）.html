新建付款申请单（草稿）

接口调用地址

生产环境地址：https://api.cmbchina.com/itrip/xft-api/v1/bills/pay-apply/create/draft

测试环境地址：https://api.cmburl.cn:8065/itrip/xft-api/uat/v1/bills/pay-apply/create/draft

接口描述

支持系统调用接口新增付款申请单，支持创建草稿和审批完结状态的付款申请单

接口输入参数

参数类型必填说明示例
headerOBJECT请求头根节点
参数类型必填说明示例
bodyOBJECT是请求体根节点
body.eventNumberSTRING是单据表单编号01240109103725000001
body.empNumberSTRING是申请人薪福通花名册员工号11111111
body.billCompanyCodeSTRING否薪福通法人公司编码，不传默认为申请人所属公司；

传入的单据公司与申请人不一致时，单据部门编号必传

billDepartmentCode和billCompanyCode需要同时指定或者同时为空		COR000001
body.billDepartmentCodeSTRING否薪福通部门组织编码，不传默认为申请人所属部门；

传入的单据部门与申请人不一致时，单据公司编号必传

billDepartmentCode和billCompanyCode需要同时指定或者同时为空		0000001
body.customFieldValuesARRAY否自定义字段对象，草稿接口允许空
body.customFieldValues.fieldNumberSTRING需要在表单配置对应的自定义字段，随机生成字段编码并保存后才可以使用

外层customFieldValues不为空时该字段非空		MULTI_INPUT_170948711
body.customFieldValues.fieldValueSTRING请参考接口描述中自定义字段传值格式和说明多行文本
body.billPayApplyOBJECT是付款申请组件
body.billPayApply.payAccessoriesSTRING否附件上传接口返回的附件自增ID

免审接口，且表单配置要求附件必填时非空		11111
body.billPayApply.remarkSTRING否免审接口，且表单配置要求备注必填时非空这是付款申请组件里的备注
body.billPayApply.payAccountOBJECT否付款申请单表单配置中，付款账户和付款方式维护模式分为制单人填写和财务人员填写

配置为制单人填写时，表单上会有公司付款账户，此时如果不传payAccount，会塞默认的账户，请提单前注意核对

配置为财务人员填写时，表单上没有公司付款账户，此时即使传入payAccount，回填仍为payAccount:null
body.billPayApply.payAccount.acctNoSTRING否公司付款账户的银行卡号
如果传银行卡号或者开户名，则两个都必须传

配置不为财务人员填写，且外层payAccount信息不为空时，该字段非空		1111111111111111
body.billPayApply.payAccount.depoNameSTRING否公司付款账户的开户名
如果传银行卡号或者开户名，则两个都必须传

配置不为财务人员填写，且外层payAccount信息不为空时，该字段非空		这是开户名
body.billPayApply.payApplyDetailListARRAY否付款申请明细

除非传值null或者[]，否则只要有字段传入，就会触发校验

校验包含：

供应商是否存在，是否适用当前单据公司

传入的付款方式中是否存在启用的对公的付款方式

付款金额是否为负数

供应商是否绑定收款账户，收款账户信息是否与供应商信息匹配

每条明细中的付款方式是否为存在的对公的付款方式
body.billPayApply.payApplyDetailList.payAmtBIGDECIMAL否最多9位整数，2位小数

无付款金额时，传0兜底		111.11
body.billPayApply.payApplyDetailList.payTypInfOBJECT否与付款账户payAccount是否非空保持一致
body.billPayApply.payApplyDetailList.payTypInf.payMethodSTRING否付款方式需要传启用的对公的付款方式，如果要传值就必须不为空

与付款账户payAccount是否非空保持一致		付款方式
body.billPayApply.payApplyDetailList.planPayDateDATE否日期格式yyyy-MM-dd2024-05-26
body.billPayApply.payApplyDetailList.supIdSTRING否供应商新增、维护时设置

外层payApplyDetailList不为空时该字段非空		SUP11111111111111
body.billPayApply.payApplyDetailList.receiveAccountOBJECT否收款账户
body.billPayApply.payApplyDetailList.receiveAccount.acctNoSTRING否付款申请明细中的收款账户银行卡号

外层receiveAccount不为空时该字段非空		1111111111111111
body.billPayApply.payApplyDetailList.receiveAccount.depoNameSTRING否付款申请明细中的收款账户开户名

外层receiveAccount不为空时该字段非空		这是开户名
body.billPayApply.payApplyDetailList.remarkSTRING否付款明细里的备注这是付款明细里的备注
body.outRelIdSTRING外部单据编号66666
body.checkDuplicateOutRelIdBOOLEANtrue:校验， false：不校验， 不传值时默认falsetrue
body.billApplyAuthCheckTypeSTRINGY需要校验 N 不需要校验
只有免审批单据才可以传N不校验		Y
